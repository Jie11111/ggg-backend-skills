# Code Review Risk Checklist

本清单只把项目约定和 `ggg-java-coding-standard` 映射到本轮真实 diff。不要重复编码规范，也不要固定填写与本轮无关的检查项。

## 1. 评审边界

- 先根据权威输入和原始 diff 形成结论，再读取 implementation 自检交叉核对。
- 评审本轮全部 diff 及真实影响面；公共契约、数据、配置和异步入口变化不能停在直接调用方。
- 每个正式问题绑定紧凑文件行号；全局契约、运行或验证问题绑定文档、命令或响应证据。
- 不把个人偏好和无关历史坏味道包装成必须修。

## 2. Gate A：Spec Compliance

Gate A 检查行为和证据，不检查代码风格：

| 检查面 | 通过条件 |
|---|---|
| 完整性 | 每项需求、验收和任务都有实现落点，或有确认的不适用依据 |
| 正确性 | 主链路、规则、状态、身份、权限、错误和副作用符合确认口径 |
| 范围 | 每个 diff 文件都能映射需求/任务；没有夹带行为和不可评审噪音 |
| 契约 | API、SQL、配置、消息及上下游兼容性与权威工件一致 |
| 证据真实性 | 文件、命令、响应和运行证据支持结论；未调查不冒充不存在 |

任一 diff 文件未覆盖，或关键结论只有 implementation 自检而无原始证据，Gate A 不得通过。

## 3. Gate B：核心质量面

每轮都检查以下六面，但只记录有信息量的证据：

| 核心面 | 检查重点 |
|---|---|
| 正确性 | 控制流、空值、边界、幂等、并发及失败后的最终状态 |
| 安全与权限 | 身份来源、越权、输入信任、敏感信息、注入及安全默认值 |
| 数据一致性 | 事务边界、写入顺序、重复消费、补偿和部分失败 |
| 失败边界 | 异常分类、重试、降级、超时、日志与定位上下文 |
| 兼容性 | 调用方、历史数据、默认值、序列化和灰度期间新旧版本 |
| 验证充分性 | 风险是否有自动化测试、运行证据或明确且合理的豁免 |

## 4. 按 diff 触发的风险模块

只加载真实触发的模块，并在 Review 明细记录触发证据：

| 模块 | 触发信号 | 重点 |
|---|---|---|
| API/契约 | Controller、Facade、DTO、序列化、错误码 | 方法、字段、身份、权限、兼容调用方 |
| SQL/事务 | Mapper、SQL、写链路、事务注解 | 隔离、索引、锁、批量、回滚和历史数据 |
| DDL/迁移 | schema、字段、索引、数据修复 | 方言、默认值、在线变更、回填、恢复 |
| MQ/异步 | 生产消费、事件、定时任务 | 幂等、顺序、重试、死信、重复和最终一致 |
| 缓存/ES | key、TTL、索引、同步链路 | 失效、穿透、旧数据、重建和一致性窗口 |
| 配置/依赖 | 配置项、开关、SDK、版本 | 默认值、环境差异、回滚、许可证和供应链 |
| 安全/权限 | 鉴权、租户、数据隔离、敏感数据 | 正负向路径、最小权限、审计和脱敏 |
| 性能/可观测 | 循环远程调用、批处理、日志/指标 | 复杂度、容量、超时、告警和 Trace |
| 前端/构建 | UI、类型、构建和静态资源 | 契约、权限展示、formatter、lint、类型检查 |

Java 的注释、日志、分层和格式仅在对应 diff 或风险触发时按共享规范检查，不再作为固定 11 项表格。工具可自动得出的事实直接记录命令结果。

## 5. 测试风险判定

- 核心业务规则、状态迁移、权限、公共契约、事务、异步副作用和缺陷回归：需要自动化测试，或记录可审计的豁免理由。
- 注释、格式、生成文件及明确无行为变化的机械调整：formatter、lint 或等价静态验证可以充分。
- Review 只判断证据是否充分，不在本阶段修改测试；缺失时登记 CRxx 并返回 implementation。

## 6. 问题级别

- `阻塞`：关键业务口径或运行事实无法确认，或存在不可接受的安全/数据风险。
- `必须修`：会造成错误行为、越权、数据不一致、兼容破坏、明显性能故障或关键验证缺失。
- `建议修`：不阻断本轮行为，但会显著降低维护或后续修改安全。
- `提示`：供后续关注，不包装成强制规范。
