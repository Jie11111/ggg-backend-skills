# 契约与接口明细检查

新增或修改 HTTP、RPC、MQ、Job、内部方法契约时读取。只在触发条件成立时创建 `interface-details/`，不要为简单内部调用增加文档。

## 主方案统一口径

在 `02-design.md` 写清：

- 契约类型：HTTP / RPC / MQ / Job / 内部方法。
- 调用方或触发源。
- 规范化契约标识；HTTP 使用 `METHOD /path`，其他类型使用方法、Topic/Event 或 Job 标识。
- 关键输入、可信来源、后端推导和禁止外部传入的字段。
- 输出字段和具体副作用；确无副作用时写“无：具体原因”，不能写裸“无”。
- 鉴权、幂等、状态隔离、异常和兼容要求。
- 来源 Cxx 和对应 Dxx。

不存在持久业务实例时明确写原因，不虚构业务实例 ID 或生命周期；确有幂等要求时使用真实请求键、消息键或任务批次键。

## 各类契约重点

- HTTP：请求方式与路径、鉴权、参数来源、请求/响应兼容、重复提交。
- RPC：方法与 DTO、调用方、超时/异常、接口演进和 Provider/Consumer 兼容。
- MQ：Topic/Event、消息键与消息体、顺序、消费幂等、重试和失败处理。
- Job：触发方式、处理范围、游标/分片、并发锁、幂等、重跑和补偿。

## 创建接口明细的条件

满足任一条件时创建明细：

- 第三方、跨团队或公开调用的外部/公共契约。
- 修改已有公共请求、响应、DTO、事件或兼容口径。
- 载荷嵌套复杂、状态分支较多，或存在关键幂等、安全、异步副作用和多类失败处理。

简单内部 HTTP/RPC、单一事件消费或普通 Job 直接留在主方案。

## 明细内容边界

- 只展开主方案放不下的契约细节，不重复背景、决策取舍和完整调用链。
- HTTP 或真实线协议给出真实请求/响应示例；RPC 或内部 Java 方法如使用共享明细模板，JSON 只作为 DTO 结构示例并明确“非线协议”，不得伪装成真实传输格式。
- 只有复杂处理分支才画接口流程图。
- 写接口或有副作用的消费/任务必须说明鉴权、幂等、状态隔离、失败和重试。
- 主方案与明细的 Dxx/Cxx、调用方、契约类型、契约标识、输入字段、后端推导字段/来源、禁止外部传字段、输出字段和副作用必须逐项精确一致。
- MQ、Job 或内部方法无同步响应时，响应表写“`不涉及：具体原因`”，响应 JSON 使用 `null` 或空集合；主方案输出字段写“无”，仍须写清真实副作用或“无：具体原因”。

## 主时序图触发

仅在以下情况画主时序图：

- 跨两个以上后端系统或第三方。
- 存在 MQ、异步回调、补偿或最终一致性。
- 存在复杂事务、跨系统状态推进或关键失败分支。

普通单服务链路用调用链和事务/异常说明即可。
