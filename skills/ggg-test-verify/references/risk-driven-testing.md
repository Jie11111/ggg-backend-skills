# 风险驱动测试参考

本参考只在制定测试计划时读取。它用于选择最低必要覆盖和执行顺序，不用于机械增加测试数量。

## 风险判断

分别判断：

- 影响严重度：失败是否造成越权、数据损坏、不可逆状态、资金/合规风险、跨系统错误或核心链路不可用；
- 发生可能性：变更是否触及复杂分支、共享状态、并发、重试、历史数据、弱契约或低可观测依赖；
- 可探测性：失败是否能被接口结果直接发现，还是只能从数据库、消息、缓存、日志或下游结果观察。

满足任一条件通常定为高风险：

- 权限、身份隔离、安全边界；
- 核心业务规则或不可逆状态迁移；
- 公共接口、共享枚举、跨仓契约；
- SQL Schema、事务边界、批量数据迁移；
- MQ、异步任务、重试、幂等或并发；
- 外部系统副作用；
- 失败不易直接观察，或历史上发生过同类缺陷。

## 最低覆盖矩阵

| 变更类型 | Implementation 应提供 | formal-gate 最低验证 | 重点观察 |
|---|---|---|---|
| Bug、核心规则 | 失败复现与回归测试；无法自动化时记录豁免 | 原缺陷复现、修复路径、相邻规则回归 | 规则结果和错误副作用 |
| 状态迁移 | 正向、拒绝路径测试 | 允许与非法转换、重复操作、最终状态 | 部分成功、状态不同步 |
| 权限与隔离 | 允许/禁止角色测试 | 正向权限、越权、跨实例数据隔离 | 未授权读写和信息泄露 |
| API/DTO/共享契约 | 契约或兼容测试 | 新旧调用、必填/默认值、错误语义 | 调用方兼容和字段丢失 |
| SQL/事务/迁移 | 集成测试或可复跑数据验证 | 提交/回滚、历史数据、约束和查询结果 | 脏数据、部分提交、性能退化 |
| MQ/任务/缓存/ES | 集成或失败路径测试 | 事件内容、次数、幂等、重试、最终一致性 | 重复消费、丢消息、陈旧数据 |
| 安全重构 | 行为特征或回归测试 | 改前改后一致性及影响面回归 | 隐式副作用和兼容性 |
| 注释/格式/机械生成 | formatter、lint 或结构校验 | 确认无行为 diff；通常无需独立业务测试 | 意外生成或范围扩张 |

## Effect

- `read-only`：只读查询、日志观察、静态检查；
- `local-write`：只生成本地测试产物；
- `data-write`：创建或修改业务数据；
- `state-change`：改变业务状态；
- `message-or-job`：发送消息、触发任务或异步消费；
- `external-side-effect`：调用真实外部系统或产生不可本地撤销的影响。

高于 `local-write` 的 effect 必须明确目标、范围、授权和恢复/回查方案。无法安全准备时标记阻塞，不绕过环境约束。

## 执行顺序

按依赖和风险排序：

1. 先验证环境、版本、数据身份是否能支撑结论；
2. 再验证最高风险且会影响后续证据有效性的业务行为；
3. 然后验证关键规则、权限、失败边界和副作用；
4. 最后执行与真实影响面匹配的兼容回归。

如果前置失败使后续结果失去意义，停止后续执行并记录阻塞。不要为了填满报告继续制造无效证据。

## 豁免

豁免不是“未执行”的别名。仅当风险已被其他证据充分覆盖，或执行成本/环境限制与风险不相称且负责人明确接受时使用。至少记录：

- 被豁免的 TBxx 和验证类型；
- 无法执行或无需执行的事实依据；
- 替代证据；
- 授权者和时间；
- 剩余风险及后续触发条件。
