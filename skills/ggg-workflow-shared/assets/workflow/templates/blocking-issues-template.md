# 阻塞问题清单模板

> 用途：这是需求对齐阶段必须单独维护的产物。没有清空前，禁止进入技术方案阶段。

## 1. 使用规则

- 只记录真正阻塞方案推进的问题
- 每个问题必须带当前判断和建议
- 每个问题都要写清当前卡点和需要谁确认
- 所有问题确认完成后，才能进入技术方案阶段

## 2. 阻塞问题

| 编号 | 问题 | 当前判断 | 推荐方案 | 备选方案 | 风险 | 工作量 | 当前卡点 / 需要谁确认 | 是否已确认 |
|---|---|---|---|---|---|---|---|---|
| B1 |  |  |  |  |  | 低 / 中 / 高 |  | 是 / 否 |

## 3. 清空标准

- 所有阻塞问题 `是否已确认=是`
- 不再新增影响方案走向的新阻塞问题
- `meta.json.blocking_issue_count=0`
