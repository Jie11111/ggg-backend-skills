## 5. 完成定义

- 所有任务都产生明确的仓库改动，并能直接交给 implementation 执行。
- 总览只维护编号、开发任务、所属项目和依赖任务；详情不重复所属项目和依赖。
- 每个任务详情都有真实来源、代码落点、主要实现内容、代码边界和可判断的完成标准。
- 每个任务至少引用一个核心改动 Dxx，且核心改动中的每个 Dxx 均已被任务覆盖。
- 总览与详情任务一致，编号唯一，依赖存在且无环。
- 跨项目改动已分开为不同 Txx，任务按纵向业务能力拆分而非机械按层拆分。
- 接口明细、`sql-draft.sql`（或旧需求的 `04-schema.sql`）及对应 Mapper/Repository/Entity 等持久化代码均已落到同一或明确关联的编码任务。
- 默认没有新增或修改测试类；用户明确要求测试代码时，已并入对应功能 Txx 并记录授权来源。
- 每个 Txx 的完成标准都包含与风险相称的最小验证。
- 未生成发布、回滚、人工 DDL、环境操作、测试执行或测试报告任务。
