# 文档权责说明

## 3 条判断规则

### 1. 需求目标以用户最新说法为准

优先级：用户最新说法 > PRD 原文 > 默认假设

### 2. 系统现状以代码证据为准

优先级：已确认代码证据 > 用户描述 > PRD 对现状的表述 > 默认假设

### 3. 阶段推进以"用户明确要求 + validate 通过"为准

必须同时满足：用户明确要求进入下一阶段 + 当前产物通过 validate。

## 权责表

| 文档 | 职责 | 不应该做什么 |
|---|---|---|
| `00-baseline.md` | 需求理解、范围边界、疑问确认记录 | 不替代代码调研结论 |
| `01-research.md` | 代码证据、主链路、可复用性结论 | 不替代技术方案 |
| `02-design.md` | 技术方案、表设计、接口、时序图 | 不重写需求事实 |
| `interface-details/` | 单接口详细设计 | 不重新定义全局方案 |
| `04-schema.sql` | 表结构变更 SQL | 不单独定义业务语义 |
| `03-tasks.md` | 基于方案的执行任务 | 不重新定义方案 |

## 冲突处理

- 下游文档与上游冲突时，以上游为准
- 先回到上游权威文档确认，再更新下游
