# 问题输出模板

## 提问原则

每次只问一个可独立决定的变量，并给出推荐，帮助用户快速拍板。用户可见内容只保留两项，推荐不能代替用户确认。

## 推荐格式

每个问题包含：

- **当前疑问**：用一两句话交代必要上下文，只描述一个不确定变量，并直接问用户
- **推荐理解**：对当前问题给出一个明确建议；依据不足时写“暂无可靠推荐”

来源、影响、推荐依据和必要的代码证据记录在疑问账本中，不默认展示给用户。需求受理阶段优先根据需求材料推荐；现有系统事实会直接影响理解或推荐可靠性时，可以只读查看相关代码，但必须牢记“代码现状不等于用户需求”，不得为了推荐提前形成表、接口或其他技术方案。推荐没有获得用户确认前，保持为建议，不得写成正式需求结论。

示例：

- **当前疑问**：需求只说明学习包下架后不能继续获取，没有说明已有用户的处理方式。已经获得该学习包的用户是否还能继续学习？
- **推荐理解**：已获得学习包的用户仍可继续学习；下架只限制新用户继续获得。
